# Publication and Quality Review

## Before Sharing Screenshots

- Confirm permission to publish institutional logos and embedded images.
- Check whether faculty names, classroom details or internal timetable links are visible.
- Remove or mask private workspace, dataset, report and local-path identifiers.
- Verify that every KPI uses the intended aggregation.
- Confirm the active school, department, programme, year and division filters.

## Before Making Numerical Claims

The PBIX uses a remote semantic model. Open the report with authorised access, record the refresh date, and document the exact slicer context. Do not publish numbers based only on report metadata.

## Recommended Portfolio Improvements

- Export one privacy-reviewed screenshot from each page.
- Add a concise data dictionary supplied by the dataset owner.
- Document the intended audience and refresh frequency.
- Add a short screen recording showing the `MAIN` → `TIMETABLE` → `ANALYSIS` journey.
