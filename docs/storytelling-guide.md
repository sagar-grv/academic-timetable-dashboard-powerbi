# Dashboard Storytelling Guide

## Core Narrative

Use the report as a three-act story rather than presenting charts randomly.

### Act 1 — Orientation

Open on `MAIN`. Explain that the report serves both timetable users and institutional decision-makers, then move to the timetable lookup.

### Act 2 — Timetable Retrieval

On `TIMETABLE`, demonstrate how the user narrows the context using School, Department, Programme, Year and Semester/Division. Explain the KPI cards and timetable link only after the filter context is visible.

### Act 3 — Workload Explanation

On `ANALYSIS`, answer five management questions:

1. How is activity distributed across departments?
2. What is the balance between theory and practical hours?
3. Which laboratories are represented?
4. How is faculty strength distributed by division?
5. Who carries class-in-charge responsibility?

## Evidence-Based Demonstration Rules

- State the active filters before describing a visual.
- Use “the report shows” only for values visible in Power BI Desktop.
- Do not infer staff performance or institutional quality from workload counts alone.
- Capture screenshots only after checking logos, names and institutional permissions.
- Keep the repository hash visible as proof that the reviewed file is the uploaded artefact.
